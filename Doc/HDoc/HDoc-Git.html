Full Playlist

Episode 1

checkout

Digunakan untuk kembali ke versi sebelumnya atau ke commit yang sebelumnya pernah dilakukan atau ke branch lain. untuk kembali ke versi/commit sebelumnya, gunakan perintah :

git checkout "5 angka hash"

Angka hash tersebut berasal dari perintah "git log". git log untuk spesifik mencari file/hal tertentu gunakan --, git log -- namaFile atau namaString

menambahkan branch, dengan perintah : git branch (nama_branch_baru)

pindah branch, git checkout (nama_branch)

Episode 6

Merge: 1. Fast Forward Merge, 2. Three-way Merge

Fast Forward: Direct Path dari branch yang mau dimerge

perintahnya: git merge namaBranch

menghapus branch, perintahnya adalah : git branch --merged untuk mengetahui branch yang sudah dimerge dan git branch -d namaBranch untuk menghapus branchnya

git branch -D namaBranch : untuk menghapus branch tanpa merging branch

Three-way Merge

Merge ini terjadi jika, branch (master) yang ingin dimerge tidak direct path

perintahnya menggunakan: git merge staff -m "pesannya", atau jika tanpa -m "pesan" maka nanti akan otomatis membuka file editor untuk menuliskan pesan commit. karena merge tipe ini akan melakukan commit, sehingga membutuhkan pesan commit.

Episode 7

Merge Conflict: ketika ada 2 branch yang mengerjakan repo/file yang sama dan saling tumpang tindih (conflict) perubahanannya

Perintah untuk merge sama saja, tetapi jika nanti ada conflict, secara otomatis file editor akan dibuka untuk menampilkan baris-baris yang conflict. pada file editor tersebut hanya perlu untuk merapihkan yang diinginkan dan menghapus yang tidak jadi dipakai dan juga menghapus marker yang dibuat sebagai tandaan dari conflict branch

Pindah ke commit dengan git checkout (7 hash number)

ketika kita pindah ke commit yang terdahulu, secara otomatis HEAD yang menujuk ke branch akan detach, kemudian untuk melanjutkan progres dari commit tersebut, perlu menambahkan lagi branch baru : git branch namaBranch

Episode 8

git cloning | cloning biasanya dilakukan ke folder yang sudah ada di remote tetapi belum ada di lokal

untuk melakukan cloning dari remote (github) gunakan perintah : git clone url

git remote

git remote -v

git config --list

git init to remote | ini dilakukan ketika sudah ada folder progress yang ada di lokal, namun belum memiliki repo di remote lakukan ini

Perlu manual untuk membuat repo di remote, tetapi untuk proses yang ini dalam pemilihannya tidak perlu membuat file readme. dan jalan kan beberapa perintah di folder lokal

git init pada folder untuk menjadikannya repo, kemudian

git remote add origin urlnya | origin merupakan default dan bisa diubah sesuai kita

setelah folder menjadi repo dan telah terhubung ke repo remote. setelahnya sudah bisa melakukan hal seperti biasa. hanya saja setelah commit ataupun akhir pekerjaan lakukan perintah

git push -u origin master | perintah ini dilakukan sekali saja, ketika melakukan di origin* dan branch* yang sama, setelahnya hanya perlu perintah git push | nama origin dan branch bisa disesuaikan.

git fetch namaRemote| namaRemote bisa opsional jika, remote yang digunakan hanya satu, jika banyak harus disesuaikan ke arah remote yang mana. digunakan untuk mengetahui perubahan commit di remote

git pull namaRemote| namaRemote -=-. digunakan untuk mengambil file yang ada di remote

git fetch dan pull sejauh ini dilakukan ketika terjadi merge conflict. tetapi mungkin bisa jadi ada kegunaan lainnya/

Episode 11

Remote Branch | Pull request ke Forked Remote Repo

Sidenote: buat branch baru untuk melakukan pull request nantinya, sehingga tidak mengganggu branch master/main

kemudian setelah membuat branch baru dan mengubah/update forked repo, kemudian commit dan push ke origin/forkingrepo (bukan repo asal).

kemudian untuk pull request lakukan pada website githubnya

kemudian untuk mengupdate yang dilokal terhadapt forked remote repo, lakukan perintah git fetch namaForkedRemote, dan perintah git merge namaForkedRemote/namaBranch untuk melakukan merge antara fetched repo dan lokal repo

git push origin --delete namaBranchBaru | digunakan untuk menghapus branch yang sudah tidak terpakai dari proses diatas tadi. perintah tersebut untuk menghapus branch yang ada di github (remote reponya) bukan lokal repo
